Le Formulaire Prendra Environ 20 mins

1. Architecture Technique

a. Technologie de Stack d'Application Non Supportée:

1. La stack technologique actuelle de votre application est-elle entièrement supportée dans votre environnement cloud cible ?




2. Quelle est la criticité de la stack technologique non supportée pour la fonctionnalité de votre application ?




3.Quel est l'effort estimé pour refactorer la stack technologique non supportée pour la compatibilité avec le cloud ?




4. Existe-t-il des alternatives commerciales ou open-source disponibles dans le cloud qui pourraient remplacer la technologie non supportée ?




5. Comment la migration de la stack technologique non supportée affectera-t-elle le coût des ressources cloud ?




b. Conception et Déploiement Monolithique:

6. Votre application est-elle conçue selon une architecture monolithique ?




7.Quelle est la complexité de décomposer l'application monolithique en microservices ?




8. Quel est l'impact de la conception monolithique sur le scalabilité de l'application dans le cloud ?




9. Comment la conception monolithique affecte-t-elle la fréquence de déploiement et l'agilité dans l'environnement cloud ?




10.Comment l'architecture monolithique affecte-t-elle l'utilisation des ressources cloud et les coûts ?




c. Points de Défaillance Unique:

11.Existe-t-il des points de défaillance unique dans l'architecture de votre application ?




12.Quel impact la présence de points de défaillance unique aura-t-elle sur la fiabilité de l'application dans le cloud ?




13.Quel est l'effort estimé pour traiter les points de défaillance unique dans votre architecture actuelle ?




14.Quelles mesures sont actuellement en place pour atténuer les points de défaillance unique ?




15.Comment les points de défaillance unique impactent-ils le plan de récupération après sinistre dans un environnement cloud ?




d. Goulots d'Étranglement:

16.Existe-t-il des goulots d'étranglement de performance connus dans l'architecture de votre application ?




17.Comment la migration vers le cloud impactera-t-elle les goulots d'étranglement de performance existants ?




18.Quel est l'effort estimé pour identifier et résoudre les goulots d'étranglement dans l'application avant la migration ?




19.Comment les goulots d'étranglement identifiés affectent-ils les métriques de performance de l'application (par exemple, temps de réponse, débit) ?




20.Quels outils ou pratiques sont en place pour surveiller et gérer les goulots d'étranglement de performance ?




e. Dépendances Cycliques:

21.Existe-t-il des dépendances cycliques dans l'architecture de votre application ?




22.Comment les dépendances cycliques affectent-elles la maintenabilité et la scalabilité de votre application dans le cloud ?

f. Automatisation des tests et couverture du code insuffisantes:

26.Quel est le niveau actuel d'automatisation des tests dans votre application ?




27.Quel est le pourcentage de couverture du code par vos tests automatisés ?




28. Quel serait l'impact d'une automatisation des tests insuffisante sur le processus de migration vers le cloud ?




29.Quel est l'effort estimé pour améliorer l'automatisation des tests et la couverture du code avant la migration ?




30. Comment les pratiques de test actuelles affectent-elles la qualité et la fiabilité de l'application dans un environnement cloud ?
